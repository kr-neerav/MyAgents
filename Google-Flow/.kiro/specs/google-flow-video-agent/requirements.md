# Requirements Document

## Introduction

The Google Flow Video Agent is an AI-powered conversational agent that helps users transform stories into 1-2 minute videos using Google Flow. Since Google Flow generates video only a few seconds at a time, the Agent guides users through the complete workflow: breaking a story into filmable sequences, identifying characters and scenes needed for visual consistency, selecting the right Google Flow capabilities for each segment, creating individual video segments, and stitching them together into a cohesive final video. The Agent outputs a step-by-step production guide tailored to the user's specific story.

## Glossary

- **Agent**: The Google Flow Video Agent AI assistant that guides users through the video creation process
- **Google_Flow**: Google's AI video generation tool that creates short video clips (a few seconds each) from text prompts and reference images
- **Story**: The narrative input provided by the user, which the Agent will help transform into a video
- **Sequence**: A discrete narrative segment of the Story that maps to one or more video clips, defined by a consistent scene, action, or dramatic beat
- **Sequence_Breakdown**: The Agent's structured decomposition of a Story into ordered Sequences with timing estimates
- **Character_Sheet**: A description of a character's visual appearance, clothing, and distinguishing features used to maintain consistency across video segments
- **Scene_Description**: A description of a location or environment including lighting, color palette, and key visual elements used to maintain consistency across Sequences set in the same location
- **Segment**: A single video clip generated by Google_Flow, typically a few seconds long
- **Production_Guide**: The Agent's complete step-by-step output document covering Sequence_Breakdown, Character_Sheets, Scene_Descriptions, prompt instructions, and stitching guidance
- **Prompt**: The text instruction and optional reference images provided to Google_Flow to generate a single Segment
- **Stitching**: The process of combining multiple Segments into a continuous final video
- **Visual_Consistency**: The maintenance of uniform character appearance, scene elements, and style across all Segments

## Requirements

### Requirement 1: Accept and Analyze Story Input

**User Story:** As a user, I want to provide my story to the Agent, so that the Agent can analyze it and plan a video production workflow.

#### Acceptance Criteria

1. WHEN the user provides a Story, THE Agent SHALL acknowledge receipt and summarize the Story's narrative arc, key characters, settings, and estimated duration as a video
2. WHEN the user provides a Story that exceeds 2 minutes of estimated video duration, THE Agent SHALL recommend cuts or condensation to fit within the 1-2 minute target and present the recommendations to the user for approval
3. WHEN the user provides a Story that is too brief for a 1-minute video, THE Agent SHALL suggest narrative expansions, additional visual beats, or pacing adjustments to reach the minimum duration
4. IF the user provides input that lacks a clear narrative structure, THEN THE Agent SHALL ask the user clarifying questions about the intended story arc, characters, and key moments before proceeding

### Requirement 2: Break Story into Sequences

**User Story:** As a user, I want the Agent to break my story into filmable sequences, so that I know exactly which video segments to create in Google Flow.

#### Acceptance Criteria

1. WHEN the Agent has accepted a Story, THE Agent SHALL produce a Sequence_Breakdown that divides the Story into ordered Sequences, each with a description, estimated duration in seconds, and the narrative purpose of the Sequence
2. THE Agent SHALL ensure each Sequence in the Sequence_Breakdown maps to a duration achievable by Google_Flow's clip length limits (a few seconds per Segment)
3. WHEN a single Sequence requires more screen time than one Google_Flow Segment allows, THE Agent SHALL subdivide that Sequence into multiple Segments with guidance on how they connect visually
4. THE Agent SHALL identify transition points between Sequences and recommend transition types (cut, fade, continuous motion) appropriate to the narrative flow
5. WHEN the user requests changes to the Sequence_Breakdown, THE Agent SHALL revise the breakdown and update all dependent timing estimates

### Requirement 3: Identify Characters and Create Character Sheets

**User Story:** As a user, I want the Agent to identify all characters in my story and create visual descriptions for each, so that I can maintain character consistency across video segments.

#### Acceptance Criteria

1. WHEN the Agent produces a Sequence_Breakdown, THE Agent SHALL identify all characters appearing in the Story and create a Character_Sheet for each character
2. THE Agent SHALL include in each Character_Sheet the character's physical appearance, clothing, distinguishing features, and any props or accessories relevant to the Story
3. WHEN a character appears in multiple Sequences, THE Agent SHALL note which Sequences the character appears in and flag any costume or appearance changes between Sequences
4. THE Agent SHALL write Character_Sheet descriptions in a format suitable for use as Google_Flow prompt references to maximize Visual_Consistency
5. WHEN the user provides reference images or specific visual preferences for a character, THE Agent SHALL incorporate those details into the Character_Sheet

### Requirement 4: Identify Scenes and Create Scene Descriptions

**User Story:** As a user, I want the Agent to identify all scenes and locations in my story and describe them visually, so that I can maintain environment consistency across video segments.

#### Acceptance Criteria

1. WHEN the Agent produces a Sequence_Breakdown, THE Agent SHALL identify all distinct locations and environments in the Story and create a Scene_Description for each
2. THE Agent SHALL include in each Scene_Description the setting details, lighting conditions, color palette, time of day, weather, and key environmental elements
3. WHEN multiple Sequences share the same location, THE Agent SHALL map those Sequences to the same Scene_Description and note any environmental changes (e.g., time progression, weather shifts) between them
4. THE Agent SHALL write Scene_Descriptions in a format suitable for use as Google_Flow prompt references to maximize Visual_Consistency
5. WHEN the user provides reference images or specific visual preferences for a scene, THE Agent SHALL incorporate those details into the Scene_Description

### Requirement 5: Recommend Google Flow Capabilities for Each Segment

**User Story:** As a user, I want the Agent to recommend which Google Flow capabilities to use for each video segment, so that I use the right tools for each part of my video.

#### Acceptance Criteria

1. WHEN the Agent produces a Sequence_Breakdown, THE Agent SHALL recommend specific Google_Flow capabilities (text-to-video, image-to-video, video extension, camera controls, style references) for each Segment
2. THE Agent SHALL explain the rationale for each capability recommendation based on the narrative requirements of the Segment
3. WHEN a Segment requires camera movement (pan, zoom, tracking), THE Agent SHALL specify the camera control parameters to use in Google_Flow
4. WHEN a Segment benefits from a reference image for visual grounding, THE Agent SHALL recommend creating or sourcing a reference image and describe what the reference image should depict
5. WHEN Google_Flow has limitations that affect a Segment (e.g., difficulty with specific actions, text rendering, or complex multi-character interactions), THE Agent SHALL note the limitation and suggest workarounds or alternative approaches

### Requirement 6: Generate Prompts for Each Segment

**User Story:** As a user, I want the Agent to write the actual prompts I should use in Google Flow for each video segment, so that I can generate video clips efficiently.

#### Acceptance Criteria

1. WHEN the Agent has completed the Sequence_Breakdown, Character_Sheets, and Scene_Descriptions, THE Agent SHALL generate a detailed Prompt for each Segment
2. THE Agent SHALL structure each Prompt to include the scene setting, character actions, camera angle, movement direction, lighting, and style descriptors in a format optimized for Google_Flow
3. THE Agent SHALL incorporate relevant Character_Sheet and Scene_Description details into each Prompt to maintain Visual_Consistency
4. WHEN the user reports that a generated Segment does not match expectations, THE Agent SHALL suggest Prompt revisions to address the specific issues the user identified
5. THE Agent SHALL number Prompts sequentially and cross-reference them to the Sequence_Breakdown so the user can track which Prompt corresponds to which part of the Story

### Requirement 7: Provide Stitching Guidance

**User Story:** As a user, I want the Agent to guide me on how to stitch all video segments together into a final video, so that I end up with a cohesive 1-2 minute video.

#### Acceptance Criteria

1. WHEN all Segments have been generated, THE Agent SHALL provide Stitching guidance including the order of Segments, recommended transition types between Segments, and timing adjustments
2. THE Agent SHALL recommend free or accessible video editing tools suitable for Stitching the Segments together
3. WHEN the user reports pacing or continuity issues in the stitched video, THE Agent SHALL suggest specific Segment re-generations, trim adjustments, or transition changes to resolve the issues
4. THE Agent SHALL provide guidance on adding audio elements (narration, music, sound effects) to complement the stitched video
5. THE Agent SHALL specify the recommended output format, resolution, and aspect ratio for the final stitched video based on the user's intended distribution platform

### Requirement 8: Output Complete Production Guide

**User Story:** As a user, I want the Agent to compile all guidance into a single step-by-step production guide, so that I have one reference document to follow through the entire video creation process.

#### Acceptance Criteria

1. WHEN the user requests the Production_Guide, THE Agent SHALL compile the Sequence_Breakdown, Character_Sheets, Scene_Descriptions, Prompts, capability recommendations, and Stitching guidance into a single structured document
2. THE Agent SHALL organize the Production_Guide in chronological production order: pre-production (characters, scenes), production (segment-by-segment prompts), and post-production (stitching, audio)
3. THE Agent SHALL include a checklist format in the Production_Guide so the user can track progress through each step
4. WHEN the user makes changes to any component (Sequences, Characters, Scenes, Prompts), THE Agent SHALL update the Production_Guide to reflect the changes
5. THE Agent SHALL include estimated time-of-effort for each step in the Production_Guide so the user can plan the video creation session

### Requirement 9: Maintain Visual Consistency Across Segments

**User Story:** As a user, I want the Agent to help me maintain visual consistency across all video segments, so that the final video looks cohesive rather than disjointed.

#### Acceptance Criteria

1. THE Agent SHALL include Visual_Consistency instructions in each Prompt, referencing the relevant Character_Sheets and Scene_Descriptions
2. WHEN generating Prompts for Segments that share characters or locations, THE Agent SHALL use identical visual descriptors for those shared elements across all relevant Prompts
3. THE Agent SHALL recommend a consistent art style, color grading approach, and aspect ratio to use across all Segments
4. WHEN the user reports visual inconsistencies between generated Segments, THE Agent SHALL diagnose the likely cause and suggest Prompt adjustments, reference image usage, or Google_Flow settings changes to improve consistency
5. THE Agent SHALL recommend generating a reference image for each major character and scene before starting Segment generation, to serve as a visual anchor for Google_Flow

### Requirement 10: Platform Deployment Guides

**User Story:** As a user deploying the Agent, I want platform-specific setup guides, so that I can deploy the Agent on my preferred AI platform.

#### Acceptance Criteria

1. THE Agent deliverables SHALL include a core prompt file named `google-flow-video-agent-prompt.md` containing the Agent's identity, methodology, and interaction patterns
2. THE Agent deliverables SHALL include a Platform_Guide named `platform-kiro.md` containing deployment instructions for Kiro CLI using the steering file approach
3. THE Agent deliverables SHALL include a Platform_Guide named `platform-claude.md` containing deployment instructions for Claude Projects
4. THE Agent deliverables SHALL include a Platform_Guide named `platform-gemini.md` containing deployment instructions for Gemini Custom Instructions and Gems
5. THE Core_Prompt SHALL be platform-agnostic, containing no references to specific deployment platforms
6. WHEN a platform has known limitations relevant to the Agent's operation, THE Platform_Guide SHALL document those limitations and recommended workarounds
