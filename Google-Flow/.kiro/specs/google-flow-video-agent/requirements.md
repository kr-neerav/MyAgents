# Requirements Document

## Introduction

The Google Flow Video Agent is an AI-powered conversational agent that helps users transform stories into 1-2 minute videos using Google Flow's video extension (extend) capability. Since Google Flow generates video in 8-second clips, the Agent guides users through the complete workflow: breaking a story into filmable sequences, identifying characters and scenes needed for visual consistency, and writing optimized prompts for video extension. The Agent can also be deployed across multiple AI platforms via platform-specific deployment guides.

## Glossary

- **Agent**: The Google Flow Video Agent AI assistant that guides users through the video creation process
- **Google_Flow**: Google's AI video generation tool that creates 8-second video clips using the video extension (extend) capability from text prompts and reference images
- **Story**: The narrative input provided by the user, which the Agent will help transform into a video
- **Sequence**: A discrete narrative segment of the Story that maps to one or more video clips, defined by a consistent scene, action, or dramatic beat
- **Sequence_Breakdown**: The Agent's structured decomposition of a Story into ordered Sequences with timing estimates
- **Character_Sheet**: A description of a character's visual appearance, clothing, and distinguishing features used to maintain consistency across video segments
- **Scene_Description**: A description of a location or environment including lighting, color palette, and key visual elements used to maintain consistency across Sequences set in the same location
- **Segment**: A single 8-second video clip generated by Google_Flow
- **Prompt**: The text instruction and optional reference images provided to Google_Flow to generate a single Segment, including visual content description, Narration text, and Background_Music suggestion
- **Narration**: Spoken text delivered by an invisible narrator accompanying each Segment, written in Hindi for 2x speed playback, conveying the Story's narrative for that portion of the video. Writing for 2x speed allows more narrative content to fit within each 8-second Segment
- **Background_Music**: A music style or track suggestion included in each Prompt to set the emotional tone for the Segment

## Requirements

### Requirement 1: Accept and Analyze Story Input

**User Story:** As a user, I want to provide my story to the Agent, so that the Agent can analyze it and plan a video production workflow.

#### Acceptance Criteria

1. WHEN the user provides a Story, THE Agent SHALL acknowledge receipt and summarize the Story's narrative arc, key characters, settings, and estimated duration as a video
2. WHEN the user provides a Story that exceeds 2 minutes of estimated video duration, THE Agent SHALL recommend cuts or condensation to fit within the 1-2 minute target and present the recommendations to the user for approval
3. WHEN the user provides a Story that is too brief for a 1-minute video, THE Agent SHALL suggest narrative expansions, additional visual beats, or pacing adjustments to reach the minimum duration
4. IF the user provides input that lacks a clear narrative structure, THEN THE Agent SHALL ask the user clarifying questions about the intended story arc, characters, and key moments before proceeding

### Requirement 2: Break Story into Sequences

**User Story:** As a user, I want the Agent to break my story into filmable sequences, so that I know exactly which video segments to create in Google Flow.

#### Acceptance Criteria

1. WHEN the Agent has accepted a Story, THE Agent SHALL produce a Sequence_Breakdown that divides the Story into ordered Sequences, each with a description, estimated duration in seconds, and the narrative purpose of the Sequence
2. THE Agent SHALL ensure each Sequence in the Sequence_Breakdown maps to a duration achievable by Google_Flow's 8-second Segment length
3. WHEN a single Sequence requires more screen time than one 8-second Segment allows, THE Agent SHALL subdivide that Sequence into multiple Segments with guidance on how they connect visually
4. THE Agent SHALL identify transition points between Sequences and recommend transition types (cut, fade, continuous motion) appropriate to the narrative flow
5. WHEN the user requests changes to the Sequence_Breakdown, THE Agent SHALL revise the breakdown and update all dependent timing estimates

### Requirement 3: Identify Characters and Create Character Sheets

**User Story:** As a user, I want the Agent to identify all characters in my story and create visual descriptions for each, so that I can maintain character consistency across video segments.

#### Acceptance Criteria

1. WHEN the Agent produces a Sequence_Breakdown, THE Agent SHALL identify all characters appearing in the Story and create a Character_Sheet for each character
2. THE Agent SHALL include in each Character_Sheet the character's physical appearance, clothing, distinguishing features, and any props or accessories relevant to the Story
3. WHEN a character appears in multiple Sequences, THE Agent SHALL note which Sequences the character appears in and flag any costume or appearance changes between Sequences
4. THE Agent SHALL write Character_Sheet descriptions in a format suitable for use as Google_Flow prompt references to maximize Visual_Consistency
5. WHEN the user provides reference images or specific visual preferences for a character, THE Agent SHALL incorporate those details into the Character_Sheet

### Requirement 4: Identify Scenes and Create Scene Descriptions

**User Story:** As a user, I want the Agent to identify all scenes and locations in my story and describe them visually, so that I can maintain environment consistency across video segments.

#### Acceptance Criteria

1. WHEN the Agent produces a Sequence_Breakdown, THE Agent SHALL identify all distinct locations and environments in the Story and create a Scene_Description for each
2. THE Agent SHALL include in each Scene_Description the setting details, lighting conditions, color palette, time of day, weather, and key environmental elements
3. WHEN multiple Sequences share the same location, THE Agent SHALL map those Sequences to the same Scene_Description and note any environmental changes (e.g., time progression, weather shifts) between them
4. THE Agent SHALL write Scene_Descriptions in a format suitable for use as Google_Flow prompt references to maximize Visual_Consistency
5. WHEN the user provides reference images or specific visual preferences for a scene, THE Agent SHALL incorporate those details into the Scene_Description

### Requirement 5: Generate Prompts for Each Segment

**User Story:** As a user, I want the Agent to write the actual prompts I should use in Google Flow for each video segment, so that I can generate video clips efficiently with rich visual content, narration, and music.

#### Acceptance Criteria

1. WHEN the Agent has completed the Sequence_Breakdown, Character_Sheets, and Scene_Descriptions, THE Agent SHALL generate a detailed Prompt for each Segment using Google_Flow's video extension capability
2. THE Agent SHALL structure each Prompt to contain sufficient visual content and action description for Google_Flow to generate an 8-second video clip, focusing on content and visuals rather than technical camera instructions
3. THE Agent SHALL incorporate relevant Character_Sheet and Scene_Description details into each Prompt to maintain Visual_Consistency
4. THE Agent SHALL include Narration text written in Hindi in each Prompt, authored for 2x speed playback, that conveys the relevant portion of the Story as spoken by an invisible narrator accompanying the video, allowing more narrative content to fit within each 8-second Segment
5. THE Agent SHALL include a Background_Music suggestion in each Prompt that sets an appropriate emotional tone for the Segment
6. THE Agent SHALL ensure each Prompt is between 200 and 500 words in length
7. THE Agent SHALL maintain a predominantly positive visual tone across all Prompts; negative characters SHALL convey negativity through facial expressions and body language rather than through frightening or scary visual appearance
8. WHEN the user reports that a generated Segment does not match expectations, THE Agent SHALL suggest Prompt revisions to address the specific issues the user identified
9. THE Agent SHALL number Prompts sequentially and cross-reference them to the Sequence_Breakdown so the user can track which Prompt corresponds to which part of the Story

### Requirement 6: Platform Deployment Guides

**User Story:** As a user deploying the Agent, I want platform-specific setup guides, so that I can deploy the Agent on my preferred AI platform.

#### Acceptance Criteria

1. THE Agent deliverables SHALL include a core prompt file named `google-flow-video-agent-prompt.md` containing the Agent's identity, methodology, and interaction patterns
2. THE Agent deliverables SHALL include a Platform_Guide named `platform-kiro.md` containing deployment instructions for Kiro CLI using the steering file approach
3. THE Agent deliverables SHALL include a Platform_Guide named `platform-claude.md` containing deployment instructions for Claude Projects
4. THE Agent deliverables SHALL include a Platform_Guide named `platform-gemini.md` containing deployment instructions for Gemini Custom Instructions and Gems
5. THE Core_Prompt SHALL be platform-agnostic, containing no references to specific deployment platforms
6. WHEN a platform has known limitations relevant to the Agent's operation, THE Platform_Guide SHALL document those limitations and recommended workarounds
